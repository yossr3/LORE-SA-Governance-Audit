# Auditing AI Decision Systems for Regulatory Compliance using Stable & Actionable Rules

## Overview

This project extends the original LORE-SA tutorial by introducing a governance audit for counterfactual explanations.

The objective is to evaluate whether the recourse generated by LORE-SA complies with ethical and data governance principles by preventing recommendations that involve protected or non-actionable attributes.

---

## Project Workflow

1. Load and preprocess the ProPublica Recidivism dataset.
2. Train a Random Forest black-box classifier.
3. Generate factual and counterfactual explanations using LORE-SA.
4. Define governance policies.
5. Audit each generated counterfactual.
6. Classify explanations as compliant or non-compliant.
